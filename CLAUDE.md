# CLAUDE.md — Pickle

> A salty web framework for sour artisans.

## What Is Pickle?

Pickle is a **code generation framework** for Go. You write controllers, migrations, request classes, and middleware — Pickle generates all the boilerplate. The output is idiomatic Go with zero runtime dependency on Pickle.

**One sentence:** Write expressive code in Go, deploy a single static binary with no runtime.

## Architecture

```
You Write (source of truth):              Pickle Generates (don't edit):
├── app/http/controllers/                 ├── app/http/pickle_gen.go
├── app/http/middleware/                  ├── app/http/requests/bindings_gen.go
├── app/http/requests/                    ├── app/models/*.go
├── routes/web.go                         ├── config/pickle_gen.go
├── database/migrations/                  └── database/migrations/types_gen.go
├── database/policies/
├── database/actions/{model}/
├── database/scopes/{model}/
├── config/
└── cmd/server/main.go
```

`pickle --watch` scans for changes and regenerates. You never edit generated files.

## What Pickle Is NOT

- **Not a runtime framework** — it's a build tool. Generated code has no dependency on Pickle.
- **Not opinionated about your database driver** — uses `database/sql`. Bring your own driver.
- **Not magic** — all generated code is visible, readable, debuggable Go.

## Design Decisions

- **Central route file** — Go has no decorators. `routes/web.go` shows the entire API surface at a glance.
- **Migrations in Go** — type-safe column definitions, schema extraction for model generation, co-located rollbacks.
- **Generated files alongside user code** — same package, `_gen.go` suffix, `// Code generated by Pickle. DO NOT EDIT.` header.
- **Query builder, not ORM** — typed query methods compiling to prepared statements. No string interpolation.

## LLM Context Efficiency

Convention over configuration means you never have to search. Validation is in request structs. Business logic is in controllers. Routes are in `routes/web.go`. Every single time.

- Controllers are pure business logic — 20 lines of intent, not 200 lines of wiring
- Request structs are self-documenting API contracts
- `routes/web.go` is the entire API surface
- Generated files never need to be read — work at the abstraction level the developer works at

### MCP Server

The Pickle MCP server gives queryable access to the project without consuming context:

- `schema:show {table}` — table structure with visibility annotations
- `routes:list` — every endpoint, middleware, request class
- `roles:list` / `roles:show {role}` — roles with permissions and column visibility
- `graphql:list` — exposed models with operations
- `make:*` — scaffold via tools instead of writing boilerplate

## Philosophy

Go's community confused *simplicity* with *tedium*. They're not the same thing. Pickle generates the boilerplate from your intent. The generated code is visible, readable, and idiomatic Go. You sacrifice nothing. You gain everything.

**Expressive DX. Go binary. No runtime.**
