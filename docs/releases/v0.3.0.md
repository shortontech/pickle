# Pickle v0.3.0

Pickle v0.3.0 adds first-party row authorization, deterministic scenario seeders, scoped resource identifiers, and stronger immutable and exported-runtime guarantees.

## Highlights

### First-party row policies and PostgreSQL RLS

- Define normalized row policies once in `database/policies/` for application enforcement on every supported database driver.
- Generate equivalent PostgreSQL row-level security for portable policies, including typed transaction-local identity, forced RLS, deterministic policy names, lifecycle state, rollback, and drift inspection.
- Derive sealed policy context automatically at HTTP and GraphQL boundaries, with trusted adapters for jobs and CLI commands.
- Inspect policy state through `policies:rows`, `policies:row`, `policies:explain`, `rls:status`, MCP tools, and affirmative Squeeze proof classifications.
- Run `pickle squeeze --live` to add explicit PostgreSQL catalog and runtime-role evidence to the static report.
- Register migration-only restrictive RLS policies as database defense in depth without allowing a manual permissive policy to broaden Pickle admission.

### Deterministic scenario seeders

- Define typed field providers, deterministic scenario graphs, stable identities, relationship propagation, repeat policies, and conflict behavior.
- Run compiled seeders with `pickle db:seed`, named scenarios, explicit seeds, dry runs, listing, and `migrate:fresh --seed`.
- Anchor deterministic fixture time with `--as-of`; immutable and append-only seed rows now derive framework UUID and integrity-chain values transactionally.
- Preserve equivalent seeder behavior in standalone exports and enforce deterministic, complete rows through Squeeze.

### Resource IDs and composite relationships

- Use the strict two-part `ResourceID` boundary type for scoped records without confusing it with an RFC UUID.
- Generate parsing helpers and Squeeze checks for UUID misuse and unscoped composite-key queries.
- Declare ordered composite primary and foreign keys across PostgreSQL, MySQL, SQLite, inspection, generation, and export.

### Integrity and generated-runtime hardening

- Append-only generated query types no longer expose update or delete operations, including standalone exports.
- Immutable reads select the globally newest version before applying row admission or soft-delete filtering, preventing older versions from resurfacing.
- PostgreSQL timestamp hashing now canonicalizes to microsecond precision on write and verification.
- Schema replay retains columns added by later `AlterTable` migrations.
- Route handler warnings inspect actual route registrations instead of unrelated composite literals.
- Exported GraphQL, immutable, append-only, policy lifecycle, identity, and RLS behavior now retain standalone parity.

## Upgrade notes

- The first `Protect(table, ...)` transition must include `ExistingRowsAlreadyValid(reason)`. Complete and verify any ownership backfill before enabling protection.
- Append-only tables cannot declare update or delete policy positions, and their generated query types intentionally omit those methods.
- Protected PostgreSQL work should use a transaction sealed with `WithPostgresPolicyContext`; Squeeze requires a checked, dominating attachment before trusting a transaction query path.
- Use structured `RestrictiveDefenseInDepth()` migration policies for manual companion RLS. Manual permissive policies on Pickle-protected tables are rejected.

## Verification

This release is covered by the complete Go test suite, standalone export compilation, generated GraphQL fixture compilation, and live PostgreSQL application-only, RLS-only, and dual-layer conformance across every portable predicate and legal operation position.
